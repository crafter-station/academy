# Crafter Academy Landing Redesign Plan V2

## Problem To Fix

The current implementation technically works, but the direction is wrong:

- The palette is too hot and red-heavy: `#ff321a` is used everywhere, with warm red panels that make the page feel aggressive.
- The page is too dense: too many bordered grids, cards, labels, metrics, and text sections compete for attention.
- The hero type is too large and all-caps, creating cropped, loud, low-legibility first impression.
- There are no image/media bays, so the page becomes a wall of typography and boxes.
- The WebGL shaders exist in `src/components/landing-atmosphere.tsx`, but they are too subtle and visually buried by overlays and the grid.

## Reference Takeaways From Dub

Use Dub as a density and hierarchy reference, not as a visual clone.

- Hero is concise: one short headline, one short supporting sentence, two CTAs.
- Trust appears early but quietly through logo/proof rows, not heavy bordered cells.
- Product/feature sections are visual-first: screenshots and UI modules explain more than paragraphs.
- Each section has one job and one clear message.
- Copy blocks are short, with generous spacing and narrow line lengths.
- Feature detail is progressively disclosed through cards, tabs, links, or visuals instead of long upfront explanations.
- Headings are large enough to scan, but not repeated at hero-level scale throughout the page.

## New Design Direction

- Keep dark mode first, but make it cooler, calmer, and more premium.
- Move from terminal/brutalist to spacious SaaS/editorial: Swiss alignment, but softer density and stronger visual rhythm.
- Use shader and media frames as the atmospheric warmth, not red panels or loud typography.
- Keep the page sellable: clearly explain the offer, fit, proof, timeline, price, and CTA with much less copy.
- Reserve real space for future images/screenshots now through designed placeholders.

## Color System

Replace the current warm red-heavy system with a cooler slate palette:

```css
--background: #05070d;
--background-soft: #090d16;
--panel: #0d1320;
--panel-raised: #111827;
--border-subtle: rgba(255, 255, 255, 0.08);
--border-strong: rgba(255, 255, 255, 0.14);
--foreground: #f8fafc;
--foreground-muted: #aab4c5;
--foreground-soft: #747f92;
--accent: #ff5a3d;
--accent-soft: rgba(255, 90, 61, 0.14);
--blue: #5b8cff;
--violet: #8b5cf6;
--cyan: #5eead4;
```

Usage rules:

- Blue, violet, and cyan carry the shader atmosphere.
- Vermilion/coral is only for the primary CTA, one or two key numbers, and focus states.
- Remove large warm red panels.
- Reduce visible grid opacity and remove most cell borders.
- Use soft panels and shadows instead of hard boxed grids everywhere.

## Typography And Spacing Rules

- Hero headline: `clamp(3rem, 7vw, 6.75rem)`, line-height `0.95`, tracking `-0.055em`, no all-caps sentence block.
- Section headlines: `clamp(2.25rem, 4.5vw, 4.5rem)`, line-height `1`, tracking `-0.04em`.
- Card headlines: `1.125rem` to `1.5rem`, line-height `1.15`.
- Body text: `1rem` to `1.125rem`, line-height `1.6`, max width `34rem` to `40rem`.
- Labels: mono, `0.72rem` to `0.8rem`, uppercase allowed only here.
- Buttons: normal readable CTA text, not ultra-spaced microcopy.
- Container: reduce from `max-w-[92rem]` to about `max-w-7xl` or `max-w-[76rem]` for most content, with full-width media only where intentional.
- Section padding: desktop `7rem` to `10rem`; mobile `4.5rem` to `6rem`.
- Target a 40-60% copy reduction.

## Copy Density Rules

- Hero headline: 5-9 words.
- Hero subhead: 18-28 words.
- Each section: one heading, one short paragraph, one visual or 2-3 concise bullets.
- Cards: one sentence only.
- FAQ: 5 questions max, answers 2-3 short sentences.
- Remove repeated explanations of the same core idea.
- Keep facts, not logistics: retain 20 seats, 10 weeks, 20 live classes, USD 999, 10 scholarships, 600+ LATAM builders, Demo Day, Launch Day, not for beginners.

## Revised Section Architecture

1. Hero
   - Eyebrow: `Cohort 01 / 20 seats`.
   - Headline option: `Build and launch with AI`.
   - Subhead: `A 10-week live academy for programmers ready to turn code into a product people can use, share, and buy.`
   - Primary CTA: `Reserve your seat` -> `/checkout`.
   - Secondary CTA: `Apply for a scholarship` -> scholarship email.
   - Compact facts row: `10 weeks`, `20 live classes`, `USD 999`, `10 scholarships`.
   - Right side: large visible shader/media frame with floating chips for `Demo Day`, `Launch Day`, and `20 seats`.

2. Trust Strip
   - One quiet line: `From the Crafter Station community: 600+ builders across LATAM`.
   - Company proof as a subdued logo/name row: Rappi, Nubank, Scotiabank, Kebo, Midudev, 30x, Clerk, Zabio.
   - No heavy bordered grid.

3. What You Build
   - Three short cards: `AI product`, `Launch story`, `Checkout-ready offer`.
   - One large placeholder visual bay for a future student product/demo screenshot.

4. Designed For Serious Builders
   - Short qualification copy.
   - Three concise checklist items.
   - Include one compact `Not for beginners` note.

5. Ten Weeks, One Launch
   - Replace the dense timeline with four milestone blocks:
     - `Weeks 1-2: Find the product`
     - `Weeks 3-4: Build the system`
     - `Week 5: Demo Day`
     - `Weeks 6-10: Price, distribute, launch`
   - Include a browser-style syllabus placeholder frame.

6. Live Cohort Experience
   - One short paragraph about live classes, feedback, and launch pressure.
   - Add a visual placeholder for community/mentor imagery or abstract cohort room.
   - Avoid WhatsApp groups, mentoring-hour mechanics, and operational details.

7. Pricing
   - One clean pricing module, not a dashboard card wall.
   - Show `USD 999`, `20 seats`, `10 scholarships available`.
   - Primary CTA remains `/checkout`; scholarship remains secondary.

8. FAQ
   - Keep only:
     - `Is this for beginners?`
     - `What will I build?`
     - `How technical is it?`
     - `What does it cost?`
     - `Are scholarships available?`

9. Final CTA
   - Calm close: `Build something that leaves the editor.`
   - One sentence and one primary CTA.

## Image And Media Placeholder Strategy

Add designed placeholder components now so the layout has room for future visuals:

- `ShaderShowcase`: large hero panel powered by WebGL, with subtle floating labels.
- `ArtifactPreview`: mock student product/demo frame using cards, screenshots slots, and status chips.
- `SyllabusPreview`: browser-like panel with week rows and milestone markers.
- `CohortPreview`: abstract mentor/community image bay using avatar placeholders and light gradients.

Rules:

- Placeholders must look intentional, not like empty gray boxes.
- Use gradients, glass panels, minimal UI primitives, and soft inner shadows.
- Mark purely decorative placeholders with `aria-hidden`.
- Do not add real image assets yet unless explicitly provided later.

## Shader Plan

The current WebGL code already has shaders, but the visual output is too subtle. Make shaders visible and useful.

Modify `src/components/landing-atmosphere.tsx`:

- Add props such as `variant="background" | "showcase"` and optional `className`.
- Keep a subtle full-page background mode if useful.
- Add a contained showcase mode for the hero visual frame.
- Increase particle count from `420` to roughly `900-1200` in showcase mode.
- Increase point size from `1.4-4.6px` to roughly `2-9px`, capped by DPR.
- Add a stronger fragment shader field: soft radial blobs/noise-like color bands in blue, violet, and cyan with minimal coral.
- Reduce or remove the overlay in `page.tsx` that currently mutes the canvas.
- Keep `prefers-reduced-motion`, document visibility pause, DPR cap, resize cleanup, shader/program cleanup, and no new dependencies.
- If WebGL fails, show a CSS gradient fallback inside the media frame.

  - Metadata can remain unless copy changes require a title update.
