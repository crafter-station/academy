# Full Syllabus Page Plan

## Goal

Add a polished public syllabus page for Crafter Academy that turns the provided 10-week planning table into a responsive, editorial-quality product-marketing page. Preserve the current landing page visual language: neutral SaaS styling, Satoshi headings, Inter body, rounded white cards, subtle borders/shadows, `academy-grid`, `dark-grid`, and compact mono labels.

## Files

- Add `src/app/syllabus/page.tsx`
- Update `src/app/page.tsx`
- Avoid `src/app/globals.css` changes unless implementation reveals a small reusable utility is genuinely needed
- No middleware/auth changes needed; `/syllabus` should be public

## Implementation

1. Create `src/app/syllabus/page.tsx` as a server component.
2. Export page metadata:
   - `title: "Syllabus | Crafter Academy"`
   - `description: "A 10-week operating system for building, shipping, and launching real AI products."`
3. Store the syllabus as structured data in the page file, not duplicated JSX.
   - Include week `0` through `9`.
   - Model each week with `week`, `theme`, `asyncPrep`, `liveOne`, `liveTwo`, `privateChallenge`, and `publicChallenge`.
   - Model live sessions with title, bullets/details, and instructor where available.
   - Treat Week 0 setup tools as `Async prep`, not live class content, matching the user's do/don't guidance.
4. Build local presentational components in the same file for minimal scope:
   - `ButtonLink`
   - `SectionLabel`
   - `HeroSection`
   - `RhythmSection`
   - `SyllabusTable`
   - `MobileSyllabusCards`
   - `OutcomeSection`
   - `FinalCta`
5. Page layout:
   - Hero with `Building is cool. Shipping is everything.` as the main message.
   - Compact stats for `10 weeks`, `20 live sessions`, `private reps`, and `public shipping`.
   - Rhythm cards explaining `Async prep`, `Live 1`, `Live 2`, `Private challenge`, and `Public challenge`.
   - Full syllabus section rendered from the same data in two formats:
     - Desktop/tablet: semantic `<table>` inside a rounded white card.
     - Mobile: stacked week cards to avoid cramped horizontal scrolling.
   - Outcome section focused on shipped product, public proof-of-work, pricing/launch readiness, and product taste.
   - Dark final CTA using `dark-grid`, with links to `/checkout` and `/`.
6. Update `src/app/page.tsx`:
   - Change `syllabusHref` from the external Notion URL to `"/syllabus"`.
   - Remove `target="_blank"` from syllabus `ButtonLink` usages now that the link is internal.
   - Update the footer `Curriculum` link from `#curriculum` to `/syllabus` because no `#curriculum` section exists.
   - Preserve the homepage's existing section layout and preview visual.

## Design Details

- Keep the page mobile-first with generous vertical spacing and single-column cards by default.
- Use `md` and `lg` breakpoints for multi-column rhythm/outcome grids and the desktop syllabus table.
- Use one `h1`, then ordered `h2`/`h3` headings for accessibility.
- Use real table semantics on desktop: `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`, and `<th scope="row">`.
- Keep mobile card labels visible as text, not implied by color or layout.
- Highlight public challenges subtly because they support the core "shipping" promise.
- Use minimal interaction polish only: hover color changes, active press scale on CTAs, no unnecessary animation.
