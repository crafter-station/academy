# Visual Style Reference Plan

## Goal
- Analyze the live `https://dub.co` homepage as the visual source.
- Save a markdown Visual Style Reference document in the workspace.
- Keep the change docs-only; do not modify app code, package files, or styling files.

## Output Path
- Create `visual-style-reference-dub-homepage.md` at the repository root.
- Rationale: the repo has no `docs/` folder or established design-reference document location, and this is a standalone artifact.

## Analysis Steps
- Open `https://dub.co` in the browser and inspect the current rendered homepage.
- Review desktop and mobile viewports to identify responsive layout differences.
- Use DevTools/computed styles where practical to confirm:
  - Background, surface, border, text, muted text, and accent colors.
  - Font family/classification, weights, scale, tracking, and line-height patterns.
  - Button, card, navigation, hero, social-proof, product visual, and footer treatments.
  - Radius, shadows, gradients, dividers, icon style, annotations, and spacing rhythm.
- Treat live-site-derived values as observed on the analysis date, not official Dub brand guidelines.

## Document Structure
- Use the exact requested sections:
  - `1. Core Aesthetic`
  - `2. Color Palette`
  - `3. Typography System`
  - `4. Key Design Elements`
  - `5. Visual Concept`
- Include the required details under each section: style name, one-sentence philosophy, influences, named colors with hex codes and usage, typography hierarchy, distinctive visual techniques, conceptual bridge, element relationships, and ideal use cases.
- Use concise markdown bullets for scanability and call out contrasts/tensions explicitly.

## Verification
- Read the completed markdown file and confirm all requested sections are present.
- Confirm the color palette lists names, hex codes, usage context, and total color count.
- Confirm typography notes include headline, body/secondary, hierarchy, and special considerations.
- No lint/typecheck/build is required because the planned change is documentation-only.

## Risks
- `dub.co` is a live site and may change after analysis.
- Exact font names may be obscured by hosted/custom font loading; if not directly confirmable, document the classification and observed behavior.
- Dynamic/A-B-tested sections may differ by viewport, cookies, geolocation, or session state.
