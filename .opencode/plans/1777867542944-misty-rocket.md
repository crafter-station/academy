# Footer Legal Links Plan

## Goal

Add a footer at the bottom of the landing page with standard legal links plus related Crafter ecosystem links, and create simple legal pages so the internal links resolve immediately.

## Recommended Changes

1. Update `src/app/page.tsx`
   - Add a semantic `<footer>` after the existing final dark CTA section and before `</main>`.
   - Preserve the current visual language: `max-w-7xl`, inline Tailwind classes, subtle borders, muted text, responsive wrapping.
   - Include internal links:
     - `/terms`
     - `/privacy`
     - `/refund-policy`
     - `/contact`
   - Include outbound ecosystem links:
     - `https://crafterstation.com`
     - Crafter Research URL, to be confirmed before implementation.
     - Kevo URL, to be confirmed before implementation.
     - Any additional related links the user confirms.
   - Use `target="_blank"` and `rel="noreferrer"` on outbound links.
   - Keep the implementation local to `page.tsx`; a shared footer component is unnecessary unless the footer will be reused across multiple routes.

2. Create `src/app/terms/page.tsx`
   - Add a minimal static Terms and Conditions page.
   - Include basic sections for program/site use, payments, refunds reference, no guarantees, liability, and contact.
   - Add simple metadata export.
   - Include a link back to `/`.

3. Create `src/app/privacy/page.tsx`
   - Add a minimal static Privacy Policy page.
   - Include sections for information collected, usage, payment/third-party services, retention, and contact.
   - Add simple metadata export.
   - Include a link back to `/`.

4. Create `src/app/refund-policy/page.tsx`
   - Add a minimal static Refund Policy page.
   - Keep copy conservative and non-specific unless the exact refund terms are provided.
   - State that users should contact the team for refund or cancellation questions.
   - Add simple metadata export and a link back to `/`.

5. Create `src/app/contact/page.tsx`
   - Add a simple Contact page with the existing scholarship/contact mailto pattern if available in `page.tsx`, or a clearly named email link if confirmed.
   - Add simple metadata export and a link back to `/`.

## Open Details Before Implementation

- Confirm the exact URLs for Crafter Research and Kevo.
- Confirm any additional footer links beyond Crafter Station, Crafter Research, and Kevo.
- Confirm the preferred contact email if it should differ from the existing scholarship email used on the landing page.

## Verification

After implementation, run:

```bash
bun run lint
bunx tsc --noEmit
bun run build
```

Manual check with `bun run dev`:

- Visit `/` and confirm the footer appears below the final CTA on desktop and mobile.
- Visit `/terms`, `/privacy`, `/refund-policy`, and `/contact`.
- Confirm internal footer links navigate correctly.
- Confirm outbound footer links open in a new tab once final URLs are provided.
