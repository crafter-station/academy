# Landing Copy Refactor Plan

## Goal

Refactor the landing page copy to sell Crafter Academy as a serious USD 999 program for engineers, not as a beginner course or scholarship-led initiative.

## Files To Change

- `src/app/page.tsx`
- `src/app/layout.tsx`

## Requirements

- Target persona: engineers who already have a technical base and want to take the program.
- Frame the offer as a program, not a course.
- Do not compare the program against programming courses across the page.
- Keep the page sellable for paid prospects at USD 999.
- Keep scholarship messaging only as one FAQ entry.
- Use `Join the program` as the primary CTA wording.

## Implementation Steps

1. Update metadata in `src/app/layout.tsx`.
   - Replace `programmers` language with `engineers`.
   - Describe the offer as a 10-week live program for building, demoing, and launching an AI product.

2. Remove scholarship prominence from `src/app/page.tsx`.
   - Delete the top-level `scholarshipHref` constant if no longer used.
   - Replace the hero `Scholarships` stat with a paid-program-relevant stat, likely `Investment: USD 999`.
   - Remove scholarship CTA buttons from the pricing card and final CTA.
   - Remove the scholarship row and explanatory paragraph from the pricing card.

3. Rewrite engineer-focused positioning copy.
   - Hero: position Crafter Academy as a live program for engineers who want product judgment, feedback, launch discipline, and momentum.
   - Program section: avoid `curriculum`, `coursework`, and beginner-course framing; focus on a product loop, shipping decisions, and artifacts.
   - Fit section: replace `Not another beginner course.` with a positive program-oriented headline.
   - Fit lists: describe strong-fit engineers and not-fit prospects without using course-comparison language.
   - Outcomes and final CTA: replace `programmers` with `engineers` and reinforce paid program value.

4. Refactor pricing copy.
   - Keep `USD 999` prominent.
   - Make the value feel intentional: one-time investment, 10-week live program, mentorship/feedback cycles, Demo Day, Launch Day, and product artifacts.
   - Use `Join the program` as the checkout CTA.
   - Keep pricing section free of scholarship copy.

5. Refactor FAQ copy.
   - Keep one FAQ item: `Are scholarships available?`
   - Make it secondary: a limited number may be available, but the main program is a paid USD 999 program for engineers ready to invest in their growth.
   - Update other FAQ answers to say `engineers` and `program` instead of `programmers` and `course`.
   - Consider adding or rewording an FAQ such as `Is this a course?` only if useful, while avoiding dismissive comparison language.

## Verification

After implementation, run:

```bash
bun run lint
bunx tsc --noEmit
bun run build
```

Optionally run `bun run dev` and inspect the page on desktop and mobile to confirm scholarship messaging appears only in FAQ and CTAs consistently say `Join the program`.
